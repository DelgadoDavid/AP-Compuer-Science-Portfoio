Python Project I Love


Play the game for yourself!

This program is the game "Battleship" renivented using code developed by me and my partner Ethan Ravenstein. In this game, the user chooses coordinates for the length
of the ship and has those ships appear on a Battleship grid as 0's. The user would play against a bot that has their own grid and that grid shows any spots that the user has
hit or miss. Hits are represented by X's and misses are represented by -'s. While the user can choose where to attack on the robot's grid, the robot will choose random
places on the user's grid.

My partner and I went through several personal testing phases while developing our program, meaning we tested our program ourselves while developing it. The main strategy
we utilized to test our program was to develop the code in segments, test to see if the segment achieved the purpose we intended it to have, and if not, look back at the code
and tweak it as necessary. My partner and I had the most difficulty in having the user input coordinates in reverse order (i.e D1, C1). We decided to fix this problem by
prompting the user to “put the higher position first” and then allowing the user to re-input the coordinates correctly.

AP Create Task: Bockey


Play the game for yourself!

This program is a multiplayer hockey simulation game that I have developed for my AP Computer Science Create Task. The game is able to be played by 2-4 players, with
each player contolling an individual athlete with their own unique controls. There is a Bang energy drink (all rights reserved) power up that would increase
the speed of the player that touched the power up.

This program was created using an incremental and iterative process in which I would write a chunk of code, test it myself, and fix it if needed. One difficulty I experienced
while developing this program was being able to restart the player and puck positions after a goal was scored. In order to fix this, I decided to utilize the broadcast system
in Scratch where a message would be brodacasted and recieved. resulting in a series of code to be activated.

For this project, a partner and I were tasked with investigating a category of our choice. With the chosen category, we were to come up with multiple
questions in that field that we could answer by researching data. After extensive and thorough research, we collected enough data to answer our
aforementioned questions and create visuals representing that data. The data for our project is based in the United States in 2015.

Figure 1: Figure 1 shows the percentage of crime commited based on gender. From that figure, we can come to the conclusion that males commit more
crime than females.
Figure 2: Figure 2 demonstrates the how often a certain type of crime is committed; from this we can conclude that the most common crime in the
United States is crimes involving drug abuse.
Figure 3: Figure 3 divides the frequency of crimes committed based on race. From this figure, we can conclude that white people commit the
most crime in the United States, with African Americans coming in a close second.
